# Crazy Trout Arena — Admin (Flutter)

Нативная кросс-платформенная версия админки на Flutter/Dart — портирована из веб-версии (PWA).

## Важно: почему нужно собрать локально

Эта песочница не имеет доступа к Flutter SDK, pub.dev и Xcode — а сборка под iOS
в принципе возможна только на macOS с установленным Xcode (это ограничение Apple,
а не какого-либо инструмента). Поэтому здесь только исходный код (`lib/` и
`pubspec.yaml`) — платформенные папки `ios/` и `android/` нужно сгенерировать
у себя одной командой.

## Как развернуть проект у себя (10 минут)

1. Установите Flutter SDK: https://docs.flutter.dev/get-started/install
2. Создайте пустой проект с нужным именем пакета:
   ```
   flutter create crazytrout_admin
   ```
3. Скопируйте из этого архива **поверх** созданного проекта:
   - `lib/` (замените целиком)
   - `pubspec.yaml` (замените целиком, либо перенесите секцию `dependencies`)
4. Установите зависимости:
   ```
   cd crazytrout_admin
   flutter pub get
   ```
5. Запуск на симуляторе/устройстве:
   ```
   flutter run
   ```

## Сборка и установка на iPhone без публикации в App Store

Как обсуждали — единственный практичный способ без App Store review — **TestFlight**:

1. Понадобится аккаунт Apple Developer ($99/год)
2. `flutter build ipa`
3. Загрузить `.ipa` в App Store Connect (через Transporter или Xcode)
4. Создать сборку в TestFlight, пригласить администраторов по email/ссылке
5. Они ставят приложение TestFlight → переходят по ссылке-приглашению → «Установить»
   (один тап, без чтения инструкций)

Обратите внимание: сборки в TestFlight действуют 90 дней, затем нужно залить новую.

## Иконка приложения

Логотип уже подготовлен и лежит в `assets/icon/`:
- `icon.png` — обычная иконка (для старых Android и iOS, без прозрачности)
- `icon_foreground.png` — версия со знаком на прозрачном фоне для adaptive-иконки Android (фон настроен на кремовый `#FBF6EC`, как в приложении)

После `flutter pub get` иконка сгенерируется автоматически во все нужные
размеры и для Android, и для iOS одной командой:
```
flutter pub run flutter_launcher_icons
```
Она сама разложит файлы по `android/app/src/main/res/mipmap-*/` и
`ios/Runner/Assets.xcassets/`.

## Как получить настоящий APK без установки Android Studio

В отличие от iOS (где сборка возможна только на Mac с Xcode), Android можно
собрать где угодно — но для этого нужен Android SDK, который здесь недоступен.
Вместо локальной установки — в проект уже добавлен GitHub Actions workflow
(`.github/workflows/build-apk.yml`), который соберёт APK на серверах GitHub:

1. Загрузите этот проект в свой репозиторий на GitHub (можно приватный)
2. Откройте вкладку **Actions** → workflow «Build Android APK» запустится
   автоматически при пуше в `main`, либо нажмите **Run workflow** вручную
3. Дождитесь зелёной галочки (2–4 минуты)
4. Откройте завершённый запуск → внизу раздел **Artifacts** → скачайте
   `app-release-apk`
5. Перекиньте `.apk` на Android-телефон и установите (Android сам предложит
   разрешить установку из «неизвестных источников» при первом запуске файла)

Это бесплатно и не требует ни Android Studio, ни Google Play Console —
всю сборку делает GitHub, вам остаётся только скачать готовый файл.



- Тарифы: Стандарт 750₽ / Гостевой 500₽ / Пенсионер 0₽
- Улов: раздельные поля кг/грамм, автоматический расчёт по фиксированной цене
  породы (без варианта «Другое»)
- Способ оплаты: Наличными / Картой
- Печать чека — **два варианта**, оба уже с полноценным нативным доступом
  (в отличие от Web Bluetooth, недоступного в Safari):
  - **«Найти принтер и распечатать»** — поиск и подключение к Bluetooth-принтеру
    напрямую (пакет `flutter_blue_plus`), отправка ESC/POS-байт
  - **«Печать через AirPrint»** — рендер чека в PDF и системный диалог печати
    (пакет `printing`, на iOS это открывает AirPrint автоматически)
- Нижняя навигация: Карта / Чек / Чеки / P&L / Профиль (Карта, Чеки, P&L,
  Профиль — заглушки, как и в веб-версии)

## Структура проекта

```
lib/
  main.dart                    — точка входа
  models/                      — Client, Tariff, CatchRow, Receipt
  data/demo_data.dart          — тарифы, породы/цены, демо-клиенты
  services/
    escpos_builder.dart        — сборка байт ESC/POS для термопринтера
    print_service.dart         — печать: Bluetooth и системный диалог (AirPrint)
  screens/
    home_shell.dart            — нижняя навигация, 5 вкладок
    receipt_screen.dart        — основной экран выставления чека
    stub_screen.dart           — заглушка для разделов в разработке
  widgets/
    catch_row_tile.dart        — строка улова (порода/кг/г/сумма)
    segmented_control.dart     — переключатель (оплата, тип чека)
    receipt_result_sheet.dart  — шторка с готовым чеком и кнопками печати
assets/icon/
  icon.png                     — иконка приложения (лого Crazy Trout)
  icon_foreground.png          — версия для adaptive-иконки Android
.github/workflows/
  build-apk.yml                — автосборка APK на серверах GitHub
```

## Дальнейшие шаги (не реализовано, но стоит учесть)

- Сейчас клиенты и цены захардкожены в `demo_data.dart` — для реальной работы
  нужен backend (API), который несколько администраторов будут видеть синхронно
- UUID сервиса/характеристики Bluetooth-принтера в `print_service.dart`
  (`000018f0…`/`00002af1…`) — это распространённый профиль для ESC/POS
  принтеров, но для конкретной модели принтера может понадобиться свериться
  с её документацией
- Для iOS нужно добавить `NSBluetoothAlwaysUsageDescription` в
  `ios/Runner/Info.plist` (Flutter создаст этот файл автоматически при
  `flutter create` — впишите туда объяснение, зачем приложению Bluetooth)
